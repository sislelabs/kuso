# kuso — Deep Review (2026-05-12)

Full-stack audit across four lenses: architecture, security, Go correctness, and UX/product. Each section reports its own P0/P1/P2 findings with file:line references. A consolidated "what to fix first" sits at the bottom.

Scope: `main` @ v0.9.79. Single-tenant positioning per CLAUDE.md (one team per cluster, Coolify-style). Multi-tenant isolation issues are explicitly out of scope.

## 1. Architecture & design

### P0

#### A-P0-1. `kube` package leaks domain logic
**Where:** `server-go/internal/kube/types.go:84-129` — `PlacementMatchesNode`, `CountPlacementMatches` are pure business rules sitting in the transport package because both `projects` and `addons` "share" them.
**Why it's wrong:** `kube` should be types + GVRs + client. Domain rules dumped here because of a real import-cycle worry, but the fix is a leaf package, not collapsing layers.
**Direction:** Extract `server-go/internal/placement/` and import from both `projects` and `addons`. `kube` keeps just the struct shape.

#### A-P0-2. Handlers do business logic; node handler has no domain service
**Where:** `server-go/internal/http/handlers/kubernetes_nodes.go:67-156` (handler builds the whole node summary inline), `handlers/projects.go:302-352` (Apply handler runs parse→plan→apply→shape).
**Why it's wrong:** `KubernetesHandler` (`handlers/kubernetes.go:30-39`) holds `*kube.Client` directly — the tell that there's no service layer beneath it. Spec apply is policy work, not HTTP plumbing.
**Direction:** New `server-go/internal/nodes/` service. Move Apply's orchestration into `internal/spec`. Handler becomes `body → svc.X → JSON`.

#### A-P0-3. KusoService → KusoEnvironment field propagation is fragile
**Where:** `server-go/internal/kube/types.go:288-342`. `KusoEnvironment.Spec` mirrors 8 fields from `KusoService.Spec` (Volumes, Placement, Internal, Runtime, Command, AdditionalHosts, TLSHosts, plus envFromSecrets).
**Why it's wrong:** The server is the only thing keeping them in sync. Every new service field needs a propagation hook + backfill + chart change. The v0.7.x silent-data-loss bug on `placement` (kusoenvironments.yaml:169) was caused by exactly this. Propagation is currently scattered across `propagateDomainsToEnvs`, `propagateInternalToEnvs`, `services_ops.go:278-298`, `env_groups.go`.
**Direction:** Either (a) teach the env helm chart to read both CRs, or (b) introduce a single `resolveEnvFromService(svc, env)` chokepoint with an explicit `Resolved` sub-block on env that's marked as server-rendered.

#### A-P0-4. `KusoCron` is three different resources crammed into one CRD
**Where:** `internal/kube/types.go:549-602`. `Kind ∈ {"service","http","command"}`. Each flavor uses a disjoint subset of fields; validation is server-side only because the schema can't express "required-if".
**Direction:** Split into `KusoServiceCron` (child of a service) and `KusoProjectCron` (standalone with image+command+optional URL). One more chart, but each schema becomes honestly validatable.

### P1

- **A-P1-1.** Domain services aren't actually shape-consistent (`projects.Service` has 22 methods including children CRUD, env-groups, drift, pods; `addons.Service` is close; `builds.Service` and `crons.Service` differ again). Split `projects.Service` into `projects` / `services` / `environments` services. (`projects/projects.go:33-88`)
- **A-P1-2.** `router.go:101-456` + `Deps{}` with 24 fields is a wiring god-object. Define a `Module interface { Mount(r chi.Router) }` registry. (`internal/http/router.go:52-456`)
- **A-P1-3.** No shared DTO package between server, CLI, and web — CLI hand-rolls types (`cli/pkg/kusoApi/projects.go:24-67, 85-90`) with comments explicitly noting drift risk. Extract `server-go/api/v1/` as a no-deps Go module; codegen web types via `tygo`.
- **A-P1-4.** `addons.Service.List` bypasses the typed `list[T]` helper to do its own dynamic-client decode (`addons/addons.go:122-137`). Loses cache acceleration; add `ListKusoAddonsFiltered` to `kube/crds.go`.
- **A-P1-5.** `projects.Service` runs 3 caches + a singleflight with separate invalidation contracts (`projects/projects.go:57-88`). Manual `invalidateDescribe` at ~12 call sites. Make caching middleware or drop the describe cache.
- **A-P1-6.** Updater rolls server image but CRD schema is human-applied; no startup check that registered Go types match live CRD. (`updater/updater.go:71-200`, `hack/release.sh:30-32`) Add a startup schema digest check; readyz fails on mismatch with a loud message. Tied to the v0.7.x data-loss class.
- **A-P1-7.** Build phase/timing/metadata live on annotations because helm-operator owns `.status` (`builds/builds.go:78-98`, `handlers/builds.go:150-203`). Replace helm-operator for KusoBuild with a 200-line Go controller; reclaim the status subresource. `KusoBuildSpec.Done` (`types.go:455-464`) goes away with it.
- **A-P1-8.** `web/src/features/` covers 8 of ~24 handler areas; new work goes to `components/`. Bias new work to `features/` per CLAUDE.md step 9.

### P2

- **A-P2-1.** Belt-and-suspenders `kube.updateWithRetry` + per-service mutex (`projects/services_deltas.go:65-66`). Drop the mutex on simple paths.
- **A-P2-2.** Three-place state map (CRDs, Postgres, in-memory) with no `docs/STATE.md`. Notify/audit/buildlog retention story is invisible.
- **A-P2-3.** `KusoAddon` carries three mutually-exclusive modes (native/external/instance) validated only at the service layer. Add CEL `x-kubernetes-validations`.
- **A-P2-4.** `notify.Dispatcher` event types are stringly typed. Central `notify/events.go` registry.
- **A-P2-5.** `health.go` and inline `readyz` in router split; pick one home.

## 2. Security & tenancy

### P0

#### S-P0-1. Unauthenticated BuildKit daemon, accessible cluster-wide
**Where:** `deploy/buildkitd.yaml:49-60`, addressed from `operator/helm-charts/kusobuild/templates/job.yaml:606` as `tcp://kuso-buildkitd.kuso.svc.cluster.local:1234`.
**Threat:** Any pod in the cluster — user app, preview env, compromised addon — can submit BuildKit gRPC requests to the privileged daemon and execute arbitrary build steps with privileged-container semantics.
**Fix:** mTLS per build job, OR NetworkPolicy restricting ingress to pods labelled `app.kubernetes.io/component=kusobuild`. Long-term, rootless daemon with fuse-overlayfs.

#### S-P0-2. Label-selector injection from raw URL params into kube list calls
**Where:** `internal/secrets/secrets.go:446`, `internal/projects/services_ops.go:762`, `internal/projects/projects_ops.go:250`. Project/service names from `chi.URLParam` concatenated into selector strings via `fmt.Sprintf`.
**Threat:** A name containing `,`/`!=` can re-shape the selector at kube apiserver evaluation time and enumerate resources from other projects. Today bounded by kube naming rules at create time, but defense-in-depth missing.
**Fix:** Validate names against `^[a-z0-9][a-z0-9-]{0,62}$` at create & read. Use `labels.Set{...}.String()` instead of string concat.

### P1

- **S-P1-1.** Release-signature verification is opt-in; default-no-key path explicitly accepts unsigned manifests (`updater/updater.go:284-309, 888-893`). A compromised GH releases endpoint (BGP/DNS hijack OR repo compromise) can roll any image. Ship a hardcoded Ed25519 pubkey in the binary; env var only as override.
- **S-P1-2.** OAuth state cookie compared with `==`, not `subtle.ConstantTimeCompare` (`handlers/oauth.go:397-403`). Timing oracle is theoretical against 128-bit entropy but trivial to fix.
- **S-P1-3.** Login rate limiter is per-process `sync.Map`; resets on pod restart, multiplies with replicas (`handlers/ratelimit.go:76`). Persist to Postgres.
- **S-P1-4.** Legacy helm chart's ClusterRole still grants wildcard `pods/exec` cluster-wide (`operator/helm-charts/kuso/templates/serviceaccount.yaml:43-44`). Remove from the legacy chart; production `deploy/server-go.yaml` already correctly omits it.
- **S-P1-5.** No security headers on `/api/*` (CSP/Frame/Nosniff only applied to SPA HTML at `spa/spa.go:190-205`). CSP includes `'unsafe-eval'` + `'unsafe-inline'`. Add nosniff + no-store middleware; remove unsafe-eval.

### P2

- **S-P2-1.** JWT redirected via URL fragment (`#token=…`, `handlers/oauth.go:435-440`). Visible in browser history + analytics. Move to short-lived one-time code exchange.
- **S-P2-2.** `KUSO_NOTIFY_ALLOW_PRIVATE_IPS=true` re-enables webhook reach to RFC1918 — including kube apiserver `10.96.0.1` (`notify/notify.go:195-210`). Document scope; consider CIDR allowlist.
- **S-P2-3.** Build's `env-detect` init container runs as root without capability drops; other init containers all drop ALL (`operator/helm-charts/kusobuild/templates/job.yaml:203-270`).
- **S-P2-4.** Per-user WS slot cap is per-pod (`handlers/logs_ws.go:45-50`). Move counter to Postgres.
- **S-P2-5.** Per-build GitHub installation-token Secret lives up to 1h post-build (Job TTL). Delete immediately on pod completion in the poller.
- **S-P2-6.** `/api/status` mounted on the pre-auth public router (`router.go:193-197`). Confirm output is safe or gate it.

### What was found to be sound
JWT alg pinned to HMAC, JTI revocation works; constant-time bcrypt-on-not-found prevents user enumeration; CSRF middleware correctly checks Origin/Referer when cookie-auth is in use; GitHub webhook HMAC + delivery dedup; SSRF transport with DNS-rebind protection; build pods correctly set `automountServiceAccountToken: false`.

## 3. Go correctness & bugs

### P0

#### C-P0-1. `Rollback` silently drops decode error, returns zero-value env
**Where:** `server-go/internal/builds/builds.go:1388`.
**Bug:** `_ = runtime.DefaultUnstructuredConverter.FromUnstructured(envRaw.Object, &e)` discards error; handler at `handlers/builds.go:109-115` then ships `{}` as 200 OK. The Patch already applied; the UI just sees garbage.
**Fix:** Return wrapped error on decode failure so the handler can 500 and the UI refetches.

#### C-P0-2. Postgres provisioner discards `row.Scan` errors → unconditional `CREATE ROLE`
**Where:** `internal/addons/instance_provisioner.go:83, 95`.
**Bug:** `_ = db.QueryRow(...).Scan(&userExists)` returns nil userExists on both "no row" AND any real query error. Race on concurrent addon-create causes one CREATE ROLE to win and the other to write a conn secret with the wrong password.
**Fix:** `if err != nil && !errors.Is(err, sql.ErrNoRows) { return ..fmt.Errorf("check pg_roles: %w", err) }`.

#### C-P0-3. Invalid double-checked locking in `loadSettings`
**Where:** `server-go/internal/builds/builds.go:263-279`.
**Bug:** Mutex released between cache-check and DB read, then re-acquired. Build storm = N concurrent `GetBuildSettings` queries, defeating the cache entirely. Compare to the correct pattern in `notify.cachedNotifications:394-411`.
**Fix:** Hold lock across the read OR use the RWMutex double-check pattern from `notify`.

### P1

- **C-P1-1.** `logship.Shipper.append` falls back to `context.Background()` if called before `Run()` sets `runCtx` (`logship/shipper.go:371-381`). Race window → unbounded goroutine. Init `runCtx` in `New()` to the server's lifecycle context.
- **C-P1-2.** Instance-addon DB check uses `db.Exec(SELECT 1 ...)` as a probe; on failure, `CREATE DATABASE` is silently skipped, and `CREATE ROLE` then succeeds against a missing DB (`addons/instance_provisioner.go:80-88`). Single `QueryRow` is the idiom.
- **C-P1-3.** Backup cleanup deletions use `context.Background()` AND discard the error (`handlers/backup.go:231, 289`). On graceful shutdown the dump-data Secret leaks. Derive a short-lived child of the server's base context; log the error.
- **C-P1-4.** `ensureCloneTokenSecret` doesn't handle `Update → NotFound` by falling back to `Create` (`builds/builds.go:1420-1426`). Concurrent cancel-delete of the token secret returns 404 to the build.
- **C-P1-5.** `kubernetes_cleanup.go:67, 87` does two cluster-wide list calls (`Jobs("")`, `Pods("")`) with no label selector and no Limit. OOM-risk on the apiserver at 10k+ pods. Add `LabelSelector` + `Limit: 500` + continue-token loop.

### P2

- **C-P2-1.** `%v` instead of `%w` for inner errors at `addons/addons.go:239,252,255`, `instancesecrets/instance_addons.go:116`, `spec/spec.go:101`. Breaks `errors.Is` chains for kube errors.
- **C-P2-2.** `logship/shipper.go:174` runs all-namespace pod list with no selector every 30s. Add `LabelSelector: "kuso.sislelabs.com/project"`.
- **C-P2-3.** `notify.Emit` persist uses `context.Background()` (`notify/notify.go:253`). Intentional but interacts badly with SQLite WAL on shutdown.
- **C-P2-4.** Cache-write race (consequence of C-P0-3): older `GetBuildSettings` response can overwrite a fresher one.
- **C-P2-5.** `nodewatch/nodewatch.go:343` allocates `errors.New(...).Error()` in a hot path.
- **C-P2-6.** Several CLI subcommands print errors but return `nil` from `RunE`, exiting 0. Audit all `cli/cmd/kusoCli/*.go` `RunE` paths.

## 4. UX & product gaps

### P0

#### U-P0-1. Coolify import has no UI surface
**Where:** `cli/pkg/coolify/{api,classify,client,inventory,types}.go` is a fully built importer. No `web/` references. `web/src/app/(app)/settings/page.tsx:41-60` lists 13 settings cards — zero mention import / migrate / Coolify.
**User experience:** Marketing promise ("super easy to transfer your projects") dies on the empty-state of `/projects` (`app/(app)/projects/page.tsx:65-80`), which has one CTA: "Create your first project". To migrate, the user must discover and use the CLI.
**Fix:** Import wizard reachable from (a) `/projects` empty state secondary CTA, (b) `/projects/new` header, (c) Settings → Instance group. POST to a new `/api/import/coolify` that delegates to the existing Go package. Dry-run preview screen with per-row include/exclude before commit.

#### U-P0-2. First-run onboarding is a 4-screen dead-drop
**Where:** `projects → /projects/new → /projects/<n> → /projects/<n>/services/new` with no progress, no detection that the GitHub App isn't installed yet (`services/new/view.tsx:33-58` silently fails when there's no installation).
**Fix:** Guided `/welcome` route with 3 steps (install GitHub App → pick repo → confirm runtime). State persists in URL. Land on canvas with a first-deploy banner.

#### U-P0-3. Audit API exists; no UI
**Where:** `docs/WORKFLOWS.md:171-176` documents `GET /api/audit`. `grep -r audit web/src/app` returns nothing.
**Fix:** `/settings/activity` page with filters (project/user/action). Reuses `/settings/users/page.tsx` table shape. ~2 days against the existing API.

### P1

- **U-P1-1.** Two different save-UX patterns across the same overlay (`ServiceSettingsPanel.tsx:312-318` floating bar with ESC-discards; `EnvVarsEditor.tsx` inline button + DiffConfirmDialog; `AddonSettingsTab.tsx:271,547` sectioned buttons; `BackupsTab.tsx:420` single button). Hoist a single SaveBar into the overlay shell via `useOverlayDirty`.
- **U-P1-2.** 7 peer tabs on the service overlay with horizontal-overflow scroll; Settings (most-destructive actions) is right-most and gets cut off at 1280px (`ServiceOverlay.tsx:39-47, 339`). Pin Settings, or fold Errors+Crons into a "More ▾".
- **U-P1-3.** `/projects/<p>` empty state offers only "Add service" (`view.tsx:120-138`). Users wanting to start with a managed DB are stuck. Add secondary "Add addon" CTA.
- **U-P1-4.** Notifications bell is admin-only (`TopNav.tsx:476, 531`). Non-admin devs lose deploy feedback after closing the service overlay. Show the per-user feed for everyone; gate only the channels-config link.
- **U-P1-5.** Project cards render `liveServices/total live` in emerald regardless of state — "1 of 5 down" looks identical to "all healthy" (`app/(app)/projects/page.tsx:163-220`). Amber when partial, red when zero-live-with-services-defined.
- **U-P1-6.** `/projects/new` hardcodes `previews.enabled: true` invisibly (`new/page.tsx:51-56`). Users get unexpected preview envs on first PR. Surface a visible toggle.
- **U-P1-7.** Locked Settings cards show "admin" badge but no "request access" affordance (`settings/page.tsx:197-220`). Replace dead opacity with mailto-to-admin or visible owner.
- **U-P1-8.** Per-project Settings page exists at `/projects/<n>/settings` but isn't linked from the canvas or breadcrumb. Add a Settings entry on the canvas toolbar.
- **U-P1-9.** Logs status pill says "open/connecting" but the underlying call is one-shot tail (`features/logs/hooks.ts`, `LogStream.tsx:62-65`). Either ship the deferred websocket gateway or re-label "auto-refresh every Ns".
- **U-P1-10.** Project list cards don't surface service URLs (`app/(app)/projects/page.tsx:180-228`). 2 clicks for "open my app". Add a hover-revealed open-↗ chip when a project has one prod service.

### P2

- **U-P2-1.** Two different popover implementations in the same `TopNav.tsx` file — `ProjectPicker` uses Popover+Command (`182-257`); `EnvironmentSwitcher` rolls its own absolutely-positioned `<ul>` with manual outside-click and ESC handlers (`259-453, 311-326`). Migrate the latter to Popover; debug the underlying Popover quirks once.
- **U-P2-2.** Non-prod env banner copy uses internal vocab ("fresh addons") with no inline explanation (`projects/[project]/view.tsx:208-238`).
- **U-P2-3.** `MobileInterstitial.tsx` blocks mobile use everywhere except `/projects/<n>`, but the canvas is the *least* mobile-friendly view; the mobile-friendly `MobileIncidentView` is hidden. Flip the rule.
- **U-P2-4.** `/projects/new` is missing the Description field (`new/page.tsx:80-104`).
- **U-P2-5.** URL preview shows literal `<service>` placeholder; users don't know if it's magic syntax (`new/page.tsx:37-41`).
- **U-P2-6.** Settings → Updates is ungated for non-admins (`settings/page.tsx:55`) but the page itself is admin-only. Add `perm:` gate.
- **U-P2-7.** Canvas right-click is the only path to common actions (Add addon, Add cron, Trigger build). Add kebab on nodes OR move "Trigger build" out of right-click since it's also in the overlay header.
- **U-P2-8.** CommandPalette doesn't index env-var keys or expose actions like "Redeploy <service>" (`CommandPalette.tsx:86`). Lowest-effort productivity win in the app.
- **U-P2-9.** Service-overlay header drift chip packs 5 states into one slot (helm-error / rolling / pending-changes / pending-restart / nothing) (`ServiceOverlay.tsx:254-320`). Split into a state pill + diagnostic chip.

### What works
- Popover-only rule from CLAUDE.md is being honored; no rogue `dropdown-menu.tsx` usages outside the primitive.
- `useOverlayDirty` shape is correct; the fix is to use it consistently.
- `EmptyState` is shared; settings index keyword search is genuinely good — better than Coolify.
- Build-row UI with rollback + duration + triggered-by attribution is more honest than Coolify ships.

## 5. Cross-cutting top-10 (fix these first)

Ordered by impact × cost. P0 means ship-blocker for the stated positioning.

1. **S-P0-1 — Lock down buildkitd.** mTLS or NetworkPolicy. Single biggest cluster-wide attack surface today.
2. **C-P0-1, C-P0-2, C-P0-3 — Three latent bugs in builds + addons.** Silent rollback decode failure, addon-create races writing wrong passwords, double-check locking that defeats build-settings cache. All one-file fixes with concrete patches in §3.
3. **S-P0-2 — Sanitize name params; switch label selectors to `labels.Set`.** Defense in depth; cheap.
4. **A-P0-3 — Stop service→env field propagation.** Pick (a) helm chart reads both CRs or (b) single `resolveEnvFromService` chokepoint with a `Resolved` block. Eliminates the largest correctness risk in the codebase and unblocks future service-level fields.
5. **U-P0-1, U-P0-2 — Coolify import wizard + guided onboarding.** Together these reverse the empty-state dead-drop that contradicts the "super easy to transfer" promise.
6. **S-P1-1 — Sign releases by default.** Embed an Ed25519 pubkey in the binary. The updater is the cluster's single most privileged code path; today it trusts an unsigned GH releases endpoint.
7. **A-P0-1, A-P0-2 — Extract `placement/` + `nodes/` services.** Restores the layering CLAUDE.md claims. Mechanical refactor; unblocks future test coverage.
8. **A-P1-6 — CRD schema-digest check on server startup.** The v0.7.x silent-data-loss bug was caused by an admin forgetting to apply the CRD; refuse to start on mismatch instead of pruning fields silently.
9. **U-P0-3, U-P1-4 — Audit UI + non-admin notifications.** Both back-end is done; this is a list view + a permission tweak.
10. **A-P1-3 — Extract `server-go/api/v1/` shared DTOs.** Stops silent contract drift between server, CLI, and (via codegen) web. The CLI's `kusoApi/projects.go:85-90` already documents the drift risk.
